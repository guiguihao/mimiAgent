# Browser-Use 登录持久化补丁说明 (Patch Documentation)

## 1. 补丁背景
`browser-use` CLI 默认在未指定 `--profile` 时使用临时沙盒环境，关闭即丢失登录态。而直接使用 `--profile` 又会因为抢占本地 Chrome 的配置目录而冲突报错。
本补丁通过引入 `BROWSER_USE_USER_DATA_DIR` 环境变量，允许 `browser-use` 使用一个**独立且持久**的配置目录，实现全自动登录持久化且不与本地 Chrome 冲突。

## 2. 修改文件
文件路径：`/Library/Frameworks/Python.framework/Versions/3.14/lib/python3.14/site-packages/browser_use/skill_cli/sessions.py`
*(注：路径可能随 Python 版本或安装环境变化，可通过 `pip show browser-use` 查看包位置)*

## 3. 修改内容 (Diff)
在 `create_browser_session` 函数中，找到 `if profile is None:` 代码块，修改如下：

```python
<<<<
	if profile is None:
		return BrowserSession(
			headless=not headed,
		)
====
	if profile is None:
		import os
		kwargs = {'headless': not headed}
		custom_dir = os.environ.get('BROWSER_USE_USER_DATA_DIR')
		if custom_dir:
			kwargs['user_data_dir'] = custom_dir
		return BrowserSession(**kwargs)
>>>>
```

## 4. 如何使用
在项目根目录的 `.env` 文件中配置该变量：

```env
# 指定专用持久化目录（需为绝对路径）
BROWSER_USE_USER_DATA_DIR="/Volumes/sandiskSSD/Project/xuexi/newclaw/workspace/browser-use-config"
```

## 5. 补丁效果
- **全自动**：只需执行 `browser-use open`，无需手动导出/导入 Cookie。
- **持久化**：所有的登录信息、LocalStorage、网站设置都会保存在指定目录下，跨重启不丢失。
- **无冲突**：与你电脑上正在运行的 Chrome 互不干扰。

## 6. 维护建议
如果未来更新了 `browser-use` 包（例如执行了 `pip install --upgrade`），该补丁会被覆盖。此时只需重新按照本说明第 3 步修改 `sessions.py` 即可恢复功能。
